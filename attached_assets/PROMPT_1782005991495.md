# Agent Prompt — Build the WKRLY Group Homepage

Paste the prompt below into your AI coding agent (Claude Code, Cursor, etc.). Keep `README.md` and `WKRLY Homepage.dc.html` in the working directory so the agent can read the full spec and open the reference.

---

You are implementing the **WKRLY Group homepage** from a high-fidelity design handoff.

**Inputs in this folder:**
- `README.md` — the complete specification: section-by-section layout, exact copy, colors, typography, and spacing tokens. This is the source of truth.
- `WKRLY Homepage.dc.html` — a rendered HTML reference of the target. Open it in a browser to see exactly what to match. It uses a tiny render runtime (`support.js`) only so it previews — **do not** carry that runtime, the `<x-dc>` wrapper, or any of its scaffolding into production. Treat the rendered result as the spec, and read the inline styles for exact values.

**What to build:**
A single marketing homepage — the "Deep Navy" direction: dark navy background, teal (`#34d6a6`) accent, Sora headings + Manrope body, a centered hero with a soft animated glow, alternating navy bands, and hairline-divided card grids. Sections, in order: Nav → Hero → Value Props (4) → Thesis → What We Do (3 cards) → Throughline → Our Approach (2×2) → Why WKRLY (2×2) → CTA → Footer. All copy and tokens are in `README.md`.

**How to build it:**
1. **Detect the environment first.** If this is an existing codebase, follow its framework, file structure, styling system (Tailwind / CSS Modules / styled-components / etc.), and existing UI primitives (buttons, links, layout containers). Reuse components rather than re-styling from scratch. If there is **no** codebase, scaffold a clean modern setup — Next.js (App Router) + Tailwind CSS is a good default — and build there.
2. **Match the design faithfully** at desktop (content max-width 1240px): colors, type scale, spacing, and the hairline-gap grid treatment should match the reference closely. Use the token table in `README.md`.
3. **Make it responsive** even though the reference is desktop-only: collapse the multi-column grids (4→2→1, 3→1, 2→1) and scale headings down at tablet/mobile breakpoints per the guidance in `README.md`. Nav collapses to a menu button on mobile.
4. **Add real interaction polish:** hover states on links and buttons (~150ms transitions), `cursor:pointer`, focus-visible outlines for accessibility. Wire the CTAs to sensible routes/anchors ("Get in Touch" → contact, "See the Portfolio" → portfolio).
5. **Motion:** the hero glow breathes (opacity ~0.5↔0.85, 8s loop). Disable it under `prefers-reduced-motion: reduce`.
6. **Semantics & a11y:** use proper landmarks (`header`, `nav`, `main`, `section`, `footer`), one `h1`, sensible heading order, sufficient contrast, and aria labels on icon-only/decorative elements (the CSS-shape icons are decorative — hide from screen readers).
7. **Fonts:** load Sora + Manrope (Google Fonts link in `README.md`, or self-host).

**Constraints:**
- Keep the exact copy from `README.md` — don't rewrite headlines or body text.
- No image assets are required; the logo and icons are simple shapes. If a real WKRLY logo exists in the codebase, use it instead.
- Don't introduce new colors or fonts beyond the token set.

When done, run the dev server and confirm the homepage renders, is responsive, and visually matches the reference.
